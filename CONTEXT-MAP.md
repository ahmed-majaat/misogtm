# Context Map

This repo is treated as multi-context for engineering skills.

## Contexts

- **Product context**: `CONTEXT.md`
- **Frontend app context**: `src/`
- **Convex backend context**: `convex/`
- **Product requirements**: `docs/prd/`
- **System ADRs**: `docs/adr/` when present
- **Frontend ADRs**: `src/docs/adr/` when present
- **Convex ADRs**: `convex/docs/adr/` when present

## Reading rule

For architecture work, read the product context first, then inspect both frontend and Convex code paths that participate in the feature or workflow under review.
